## 目录定位

`docs/` 保存面向仓库维护者的长期文档，覆盖设计、开发流程、知识沉淀与参考材料。
这里的文档应反映当前仓库实际状态，而不是脱离代码的抽象说明。

## 子目录

| 目录 | 用途 | 备注 |
|------|------|------|
| `design/` | 设计文档 | 记录架构方案、设计草图与决策背景 |
| `development/` | 开发流程与指南 | 记录生成链路、开发约束、日常操作说明 |
| `knowledge/` | 知识库 | 沉淀概念说明、排障经验与长期参考信息 |
| `reference/` | 参考材料 | 放分析结果、外部资料整理、现状盘点 |
| `performance/` | 性能相关 | 放性能测试、压测结果与调优记录（如存在） |

## 编写约定

- 优先记录 **为什么这样做**、**影响哪些目录/命令**、**何时需要重新生成**，避免只写表面步骤
- 文档中涉及生成链路时，优先引用根目录命令：`make gen`、`make api`、`make openapi`、`make wire`、`make ent`
- 涉及前端时，说明 TypeScript 客户端来自 `api/gen/ts/` / `@servora/api-client`，不要写成手工维护类型
- 涉及 OpenFGA、Proto、Wire、Ent 等改动时，写清对应回放命令，避免读者遗漏再生成/同步步骤

## 何时拆分子 AGENTS

当前 `development/` 与 `reference/` 规模仍小，默认继承本文件即可。
只有在某个子目录形成了稳定且独立的维护规则（例如固定模板、专用命令、显著不同的读者对象）时，再新增子级 `AGENTS.md`。

## 维护提醒

- 修改文档后，确保内容与根 `AGENTS.md`、相关服务/模块目录的实际实现一致
- 引用路径默认以仓库根目录为基准，避免相对路径歧义
- 不要在文档中指导读者手改生成产物，如 `api/gen/go/`、`api/gen/ts/`、`wire_gen.go`、`openapi.yaml`
